# ADR 0020 – HUD (Run-Timer, Wave-Counter, Mutation-Liste)

- Status: **Accepted**
- Datum: 2026-05-06
- Entscheider: game-architect (Lead), shader-fx-specialist (konsultiert)
- Betrifft Prinzipien: #2 EventBus, #4 i18n, #7 testbar
- Voraussetzungen: ADR 0006 (RunState), ADR 0015 (PlayerMutations), ADR 0019 (GameOver-Pattern)
- Wird vorausgesetzt von: Mutation-Pick-Phase (eigenes ADR), HP-Anzeige im HUD

---

## 1. Kontext

Spiel ist seit v0.0.4 spielbar. Spieler weiß aber nicht, in welcher Welle
er ist, wie lang er schon lebt oder welche Mutationen aktiv sind. HP-Bar
über dem Mob ist da, aber Game-Information fehlt.

Anforderungen v1:

- **Run-Timer** (oben mitte): mm:ss seit Run-Start
- **Wave-Counter** (oben links): „Wave N" + ggf. Difficulty
- **Mutation-Liste** (oben rechts): Anzeige aller gepickten Mutationen
- **Reagiert auf EventBus**: wave_started, mutations_changed
- **Run-Timer pollt** RunState.get_run_time() pro Frame
- **Visibility**: aktiv während RunState.is_running, hidden bei ended
- **Headless-testbar**: Format-Helper + Update-Methoden direkt aufrufbar

Bewusst NICHT in v1:

- HP-Anzeige im HUD (HealthBar über Player reicht)
- Damage-Numbers (eigenes ADR)
- Animations / Pop-Effekte beim Wave-Wechsel
- Mutation-Tooltips (eigenes ADR)
- Pause-Menü-Integration
- i18n der Labels (Dev-Strings v1, lore-writer + i18n-ADR später)

## 2. Optionen

### Option A — CanvasLayer mit 3 Labels (empfohlen)

```
HUDOverlay (CanvasLayer)
└── Container (MarginContainer)
    ├── TimerLabel (Label, oben mitte)
    ├── WaveLabel (Label, oben links)
    └── MutationsLabel (Label, oben rechts, multi-line)
```

**Pro**
- Konsistent mit GameOver-Pattern (CanvasLayer-Overlay)
- Headless-testbar (Labels haben einfache `text`-Properties)
- Player-Welt bleibt sichtbar im Hintergrund

**Contra**
- Layout-Code ist Inspector-only — Anchors müssen sauber gesetzt sein

### Option B — Control-Hierarchy als VBoxContainer

Statt CanvasLayer: einfach ein Control-Tree direkt in RunScene.

**Pro**
- Weniger Boilerplate

**Contra**
- HUD verschiebt sich mit Camera (will man nicht)
- Nicht konsistent mit GameOver-Pattern

### Option C — In-World Text (Node2D + Label)

HUD als Welt-Element in der Scene.

**Pro**
- Künstlerisch (z.B. „diegetisches" UI)

**Contra**
- Funktioniert nicht für ein Survivor-likes (HUD muss screen-space sein)

## 3. Empfehlung

**Option A** — CanvasLayer mit 3 Labels, layer < GameOver-Layer (damit
GameOver darüber liegt).

**Begründung**
- Konsistent mit ADR 0019 (CanvasLayer-Overlay)
- Drei Labels = drei Update-Hooks, klar zu testen
- Layer-Konvention klar: HUD layer=50, GameOver layer=100

**Public-API**

```gdscript
class_name HUDOverlay extends CanvasLayer

func update_wave(wave_idx: int, difficulty: float) -> void
func update_mutations(mutation_ids: Array[StringName]) -> void
func set_run_active(active: bool) -> void   # Visibility-Toggle

# Format-Helper (testbar)
static func _format_time(seconds: float) -> String   # "1:23"
```

**Lifecycle**

```gdscript
func _ready():
    # Hooks
    EventBus.run_started.connect(_on_run_started)
    EventBus.run_ended.connect(_on_run_ended)
    EventBus.wave_started.connect(_on_wave_started)
    EventBus.mutations_changed.connect(_on_mutations_changed)
    visible = false

func _process(delta):
    if RunState.is_running():
        timer_label.text = _format_time(RunState.get_run_time())

func _on_run_started(_dino_id):
    set_run_active(true)
    update_wave(1, 1.0)
    update_mutations([])

func _on_run_ended(_reason, _run_time):
    set_run_active(false)

func _on_wave_started(wave_idx, difficulty):
    update_wave(wave_idx, difficulty)

func _on_mutations_changed():
    update_mutations(PlayerMutations.get_picked())
```

**Visual-Spec v1**

- TimerLabel: oben mitte, 24px, weiß, „0:00"-Format
- WaveLabel: oben links, 18px, „Wave 1"
- MutationsLabel: oben rechts, 14px, mehrere Zeilen mit IDs
  („triceratops_horns\nankylosaur_plates\n...")

## 4. Konsequenzen

**Positiv**
- Spieler hat Game-Information während des Spielens
- HUD wiederverwendbar in Pause-Menü, Mutation-Pick-Phase
- Tests können Format-Logik isoliert prüfen

**Negativ**
- Run-Timer pollt im _process — kein Bus-Event nötig (RunState hat
  kein ticked-Signal). Akzeptabel: 60×/s ein einfacher Float-Cast.

**Risiken**
- **Risiko:** Mutations-Liste mit 20+ Picks wird zu groß → Layout
  überflutet.
  → **Mitigation:** Auto-Scroll oder Truncate kommt mit Mutation-Pick-UI-ADR.
